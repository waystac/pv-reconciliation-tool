# 光伏结算对账核销工具

一款基于 Python 的轻量级桌面工具，用于将光伏电费结算中的汇总表与明细表自动配对勾销，大幅减少人工核对工作量。

## 功能特点

- **智能匹配策略**：依次尝试整月匹配、连续月份组合、单条匹配、动态规划（DP）精确凑数、回溯搜索，确保尽可能覆盖所有匹配场景。
- **税率自动尝试**：支持配置多种税率（0%、1%、3%、13% 等），程序自动逐一尝试，匹配成功时记录所用税率。
- **双列备用匹配**：当主明细列无法匹配时，可启用备用金额列进行二次尝试。
- **中英文括号兼容**：自动识别并忽略用户名中的中英文括号差异。
- **历史记录增量核对**：支持加载上次导出结果，已匹配的记录自动跳过，仅对新增数据进行匹配。
- **详细失败原因**：对未匹配的记录，输出每一步的失败原因，方便人工复核。
- **图形化界面**：基于 tkinter 的简洁操作界面，无需命令行知识。
- **日志导出**：支持将核对过程的详细日志保存为文本文件。

## 局限性

本工具针对光伏结算的常见场景进行了优化，但仍存在以下已知局限：

### 1. 单条 DP 金额上限（5 万元）

当目标结算金额超过 5 万元时，动态规划（DP）算法会被自动跳过，仅依赖整月匹配、连续月份匹配、单条匹配和回溯搜索。

**原因**：DP 需要创建大小为"目标金额（分）"的数组，超大金额会导致内存爆炸和计算卡顿。5 万元是一个经验性的安全阈值，在速度和匹配率之间取得了平衡。

**影响**：大额结算（如几十万元的整笔结算）如果无法通过整月或连续月份匹配，将依赖回溯（见下一条限制）或直接标记为未匹配。

**建议**：对于超大额结算，可在 Excel 中提前按月拆分汇总数据，使其降至 5 万元以下，再使用本工具核对。

### 2. 回溯条数上限（22 条）

回溯搜索（从 2 条开始递增尝试组合）最多只对 22 条明细进行计算。如果当前可用明细超过 22 条，程序仅取日期最新的 22 条参与回溯。

**原因**：回溯本质上是穷举组合，条数越多计算量呈指数级增长，限制 22 条是为了在可接受的时间内完成计算。

**影响**：如果一个结算项目对应超过 22 条明细，且整月匹配、连续月份匹配、单条匹配均失败，回溯可能因未包含正确的明细组合而匹配失败。

**建议**：对于明细条数极多的结算，建议在 Excel 中按月份拆分明细数据，或提前手工核对部分明细，减少参与匹配的条数。

### 3. 浮点精度

程序使用 Python 的 `Decimal` 类型进行高精度计算，税率换算时采用"税额先四舍五入再加原值"的方式，但与 Excel 的 ROUND/ROUNDDOWN/ROUNDUP 等不同舍入方式可能产生分位差异。

**影响**：极少情况下，程序计算的含税合计与 Excel 手工计算的结果相差 0.01~0.03 元，导致匹配失败。

**建议**：可在界面中适当增大"匹配容差"（如设为 0.05 元），以包容这种微小差异。

### 4. 重复行处理

当汇总表或明细表中存在"用户名 + 金额 + 日期"完全相同的多行记录时，历史加载的恢复顺序可能与首次匹配不一致，导致两次运行结果存在微小差异。

**建议**：尽量保证原始数据中每行记录是唯一的。如有重复，可在 Excel 中添加辅助列（如序号）区分。

### 5. 仅支持 Excel 格式

目前只支持 `.xlsx` 和 `.xls` 格式的 Excel 文件，不支持 CSV 或其他数据源。

## 安装与运行

### 环境要求

- Python 3.7 或更高版本
- 安装依赖库：
  ```bash
  pip install pandas openpyxl
  ```

### 运行方式

1. 下载本项目代码到本地。
2. 在命令行中运行：
   ```bash
   python main.py
   ```
   （请将 `main.py` 替换为实际的脚本文件名）
3. 或使用 Thonny 等 Python IDE 打开并运行。

### 打包为 exe（可选）

```bash
pip install pyinstaller
pyinstaller --onefile --noconsole --name "光伏结算对账核销工具" main.py
```

生成的 exe 文件位于 `dist` 目录下，可直接在未安装 Python 的电脑上运行。

## 使用说明

### 1. 准备数据

需要两个 Excel 文件：

**汇总表（表一）**：必须包含以下列（列名可在界面中自定义）：
- 用户名
- 总数（结算金额）
- 日期

**明细表（表二）**：必须包含以下列：
- 用户名
- 明细（不含税金额）
- 日期（格式如 `202501`）

### 2. 基本操作

1. 打开程序，点击"浏览..."分别选择汇总表和明细表文件。
2. 确认列名设置与表格表头完全一致。
3. 根据需要设置税率列表和匹配容差。
4. 点击"开始核对"，等待完成。
5. 点击"导出结果 Excel"保存核对结果。

### 3. 历史增量核对

1. 首次核对后，导出结果文件。
2. 下次核对时，勾选"加载历史匹配"，选择上次导出的结果文件。
3. 程序会自动恢复已匹配的记录，仅处理新增数据。

### 4. 列名配置

| 列名 | 说明 | 默认值 |
|------|------|--------|
| 用户名列 | 汇总表与明细表共有 | 用户名 |
| 汇总表 总数列 | 结算金额 | 总数 |
| 汇总表 日期列 | 结算日期 | 日期 |
| 明细表 明细列 | 主明细金额 | 明细 |
| 明细表 备用列 | 备用金额（可选） | 备用 |
| 明细表 日期列 | 明细月份 | 日期 |
| 明细表 税率列 | 明细自带税率（可选） | 留空 |

## 匹配逻辑

程序对每一笔汇总记录依次尝试以下策略：

1. **整月匹配**：将同一月份的所有可用明细含税总额与目标总数比较（支持容差）。
2. **连续月份匹配**：寻找连续几个月的总额之和等于目标。
3. **单条匹配**：查找是否有单条明细等于目标。
4. **单条 DP**：目标金额 ≤ 5 万元时，使用动态规划精确凑数。
5. **回溯**：从 2 条开始递增搜索组合（最多取日期最新的 22 条明细）。

若主明细列全部失败，且启用了备用列，则用备用列重复上述流程。

## 结果说明

导出的 Excel 文件包含两个工作表：

- **汇总表-核对结果**：包含匹配状态、匹配组号、所用税率、失败原因、差值等列。
- **明细表-核对结果**：包含匹配总数、匹配组号、换算税率、匹配来源等列。
