# 光伏结算对账核销工具

一款基于 Python 的轻量级桌面工具，用于将光伏电费结算中的汇总表与明细表自动配对勾销，大幅减少人工核对工作量。

## 功能特点

- **智能匹配策略**：依次尝试整月匹配、连续月份组合、单条匹配、动态规划（DP）精确凑数、回溯搜索，确保尽可能覆盖所有匹配场景。
- **税率自动尝试**：支持配置多种税率（0%、1%、3%、13% 等），程序自动逐一尝试，匹配成功时记录所用税率。
- **双列备用匹配**：当主明细列无法匹配时，可启用备用金额列进行二次尝试。
- **中英文括号兼容**：自动识别并忽略用户名中的中英文括号差异。
- **历史记录增量核对**：支持加载上次导出结果，已匹配的记录自动跳过，仅对新增数据进行匹配。
- **详细失败原因**：对未匹配的记录，输出每一步的失败原因，方便人工复核。
- **图形化界面**：基于 tkinter 的简洁操作界面，无需命令行知识。
- **日志导出**：支持将核对过程的详细日志保存为文本文件。

## 技术栈

- Python 3.x
- Pandas（数据处理）
- tkinter（图形界面）
- Decimal 高精度计算

## 安装与运行

### 环境要求

- Python 3.7 或更高版本
- 安装依赖库：
  ```bash
  pip install pandas openpyxl
  ```

### 运行方式

1. 下载本项目代码到本地。
2. 在命令行中运行：
   ```bash
   python main.py
   ```
   （请将 `main.py` 替换为实际的脚本文件名）
3. 或使用 Thonny 等 Python IDE 打开并运行。

### 打包为 exe（可选）

```bash
pip install pyinstaller
pyinstaller --onefile --noconsole --name "光伏结算对账核销工具" main.py
```

生成的 exe 文件位于 `dist` 目录下，可直接在未安装 Python 的电脑上运行。

## 使用说明

### 1. 准备数据

需要两个 Excel 文件：

**汇总表（表一）**：必须包含以下列（列名可在界面中自定义）：
- 用户名
- 总数（结算金额）
- 日期

**明细表（表二）**：必须包含以下列：
- 用户名
- 明细（不含税金额）
- 日期（格式如 `202501`）

### 2. 基本操作

1. 打开程序，点击"浏览..."分别选择汇总表和明细表文件。
2. 确认列名设置与表格表头完全一致。
3. 根据需要设置税率列表和匹配容差。
4. 点击"开始核对"，等待完成。
5. 点击"导出结果 Excel"保存核对结果。

### 3. 历史增量核对

1. 首次核对后，导出结果文件。
2. 下次核对时，勾选"加载历史匹配"，选择上次导出的结果文件。
3. 程序会自动恢复已匹配的记录，仅处理新增数据。

### 4. 列名配置

| 列名 | 说明 | 默认值 |
|------|------|--------|
| 用户名列 | 汇总表与明细表共有 | 用户名 |
| 汇总表 总数列 | 结算金额 | 总数 |
| 汇总表 日期列 | 结算日期 | 日期 |
| 明细表 明细列 | 主明细金额 | 明细 |
| 明细表 备用列 | 备用金额（可选） | 备用 |
| 明细表 日期列 | 明细月份 | 日期 |
| 明细表 税率列 | 明细自带税率（可选） | 留空 |

## 匹配逻辑

程序对每一笔汇总记录依次尝试以下策略：

1. **整月匹配**：将同一月份的所有可用明细含税总额与目标总数比较（支持容差）。
2. **连续月份匹配**：寻找连续几个月的总额之和等于目标。
3. **单条匹配**：查找是否有单条明细等于目标。
4. **单条 DP**：目标金额 ≤ 5 万元时，使用动态规划精确凑数。
5. **回溯**：从 2 条开始递增搜索组合（最多取日期最新的 22 条明细）。

若主明细列全部失败，且启用了备用列，则用备用列重复上述流程。

## 结果说明

导出的 Excel 文件包含两个工作表：

- **汇总表-核对结果**：包含匹配状态、匹配组号、所用税率、失败原因、差值等列。
- **明细表-核对结果**：包含匹配总数、匹配组号、换算税率、匹配来源等列。
